# Базы данных
**База данных** - упорядоченное цифровое хранилище информации, что бы компьютер мог быстро находить, добавлять или изменять данные. 

**СУБД**(системы управления базами даннх) - это набор инструментов, которые позволяют удобно управлять базами данных: удалять, добавлять, фильтровать, находить элементы и т.п.

**Типы баз данных:**
- Реляционные (данные хранятся в виде таблицы, которые связаны друг с другом через сквозные параметры.)
- Нереляционные (данные могут  хранится в виде документов, графов, а также ключей значений)

## Реляционные базы данных 
**Особенности**
- Жесткая (фиксированная)
- Вертикальное масштабирование
- Использует структурированный язык запросов SQL
- Обеспечивает высокую целостность данных
- Управление связями (легко определить связь и избежать дублирования)

**Примеры**
- PostgreSQL
- MsSQL
- Maria DB
- и т.д.

### PostgreSQL
**Основные особенности:**
- Свободный доступ (можно бесплатно скачать, установить СУБД и сразу начать работу с базами данных.)
- Можно установить на любую платформу (например, Linux, macOS, Windows.)
- Поддерживает разные форматы данных (например, сетевые адреса, данные в текстовом формате JSON и геометрические данные для координат геопозиций.)
- Соответсвует требованиям ACID 
  - атомарность (возможность совершать транзакции)
  - согласованность (данные остаются в целостном состоянии) 
  - изолированность (параллельные действия не влияют друг на друга)
  - устойчивость (уже выполненая транзация не отменится из-за тех.неполадок)
- Поддерживает все функции, которые есть в современных базах 
  - Оконные функции позволяют выбрать определенные записи в таблице и делать вычисления с ними в отдельном столбце
  - Вложенные функции - транзакции внутри других.
  - Триггеры - функции, которые автоматически запускаются при определенных условиях.
- Есть свой диалект яхыка SQL

### MsSQL
**Основные особенности:**
- Интеграция с экосистемой Microsoft
- Высокая производительность и масштабируемость
- Соответсвует требованиям ACID
- Богатый набор функций
  - Поддержка XML и JSON
  - Расширенные возможности безопасности ( прозрачное шифрование данных, защита на уровне строк, динамическое маскирование данных.)
- Использует диалект Transact-SQL (T-SQL) (расширение стандарта SQL с поддержкой переменных, условных конструкций, циклов, обработки ошибок и мощных возможностей для написания сложной бизнес-логики на стороне базы данных.)

### Maria DB
**Основные особенности:**
- Свободный доступ
- Можно установить на любую платформу (например, Linux, macOS, Windows.)
- Полная совместимость с MySQL
- Соответствует требованиям ACID
- Одновременные использование различных механизмов хранения (поддерживает несколько механизмов хранения, что позволяет управлять различными типами рабочей нагрузки в рамках одной СУБД.)


## Нереляционные базы данных 
**Особенности**
- Гибкая (динамическая) структура данных
- Горизонтальное масштабирование
- Использует различные языки запросов (API, JSON-запросы, языки в зависимости от типа БД)
- Обеспечивает высокую доступность и производительность
- Слабая связность данных (связи управляются на уровне приложения)

**Примеры**
- MongoDB
- ClickHouse
- Redis

### MongoDB
**Основные особенности:**
- Кросс-платформенность (работает с Windows, Linux и macOS)
- Базы хранятся в виде наборов из связанных между собой экземпляров (реплик) на нескольких серверах (узлах).
- Чтобы увеличить пропускную способность СУБД, при работе с большими данными применяется шардирование ( базы разбиваются на части (шарды) и размещаются на разных серверах.)
- Размер документов ограничен (он не может быть больше 16 МБ)
- Данные можно искать по полям и диапазонам
- Есть собственный язык запросов.
- Базы MongoDB могут работать сразу на нескольких серверах

### ClickHouse
**Основные особенности:**
- Высокая производетильность (может обрабатывать миллиарды строк за считаные секунды.)
- Эффективное использование ресурсов (Благодаря колоночной архитектуре эффективно сжимает и хранит данные, что позволяет экономить пространство на диске и использовать меньше оперативной памяти.)
- Поддержку SQL
- Высокая доступностьм (данные всегда доступны, даже если произойдут сбои в работе серверов)
- Обнаружение отказов (постоянно проверяет работоспособность своих серверов.)

### Redis
**Основные особенности:**
- Производительность (может обрабатывать миллионы запросов за секунду.)
- Работает с разными структурами данных
- Быстрее проходят операции
- Гибче работа с базой (Структуры данных более гибкие, и у разработчика больше свободы в том, как ими управлять.)
- Нет языка SQL (применяет скрипты на языке Lua, который специально спроектирован очень простым и поэтому легок в изучении.)
- Возможность делать «снимки» базы и обеспечивать персистентность, то есть фактически контроль версий.
